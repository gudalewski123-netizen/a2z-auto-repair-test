Build a lightweight CRM system for service-based businesses (starting with car detailing use case). The system should focus on capturing, organizing, tracking, and converting leads into paying customers.

CORE FEATURES:

1. Lead Capture (Form System)

* Create a public-facing form that collects:

  * Name
  * Phone number
  * Email
  * Service requested (dropdown or text)
  * Additional notes
* Form submissions should be saved to a database instantly
* Each submission should automatically create or update a contact (no duplicates based on phone/email)

2. Contact Management (CRM Core)

* Each contact should have a profile page with:

  * Name, phone, email
  * All past submissions/jobs
  * Total revenue (lifetime value)
  * Notes section (editable)
  * Activity log (timestamps of actions like created, contacted, booked)

3. Pipeline / Status Tracking

* Implement pipeline stages:

  * New Lead
  * Contacted
  * Booked
  * Completed
  * Lost / No Show
* Each contact must have a status
* Allow drag-and-drop or dropdown to change status
* Track timestamps when status changes

4. Dashboard / Admin Panel

* Clean admin UI showing:

  * All contacts in a table
  * Filters (by status, date, service)
  * Search (name, phone, email)
* Show counts:

  * Total leads
  * Booked jobs
  * Completed jobs
  * Conversion rate

5. Follow-Up System

* Allow setting manual reminders for each contact:

  * Date + note
* Show upcoming follow-ups in dashboard
* Highlight overdue follow-ups

6. Basic Automation

* When a form is submitted:

  * Automatically send SMS or email:
    “We received your request and will contact you shortly”
* Allow toggling automation on/off

7. Source Tracking

* Track where each lead came from:

  * Website
  * Google Ads
  * Facebook
  * Referral
* Store source in contact profile
* Allow filtering by source in dashboard

8. Job / Revenue Tracking

* Allow adding a job to a contact:

  * Service type
  * Price
  * Date
* Automatically calculate:

  * Total revenue per contact
  * Total revenue across all contacts

9. UI/UX Requirements

* Mobile-friendly responsive design
* Fast and minimal UI (no clutter)
* Clickable phone numbers and emails
* Clean, modern dashboard style (similar to simple SaaS tools)

10. Tech Requirements

* Use a simple backend (Node.js or Python)
* Use a database (SQLite or PostgreSQL)
* REST API structure for scalability
* Clean code structure for future feature expansion

11. Optional (if time permits)

* Tagging system (e.g. “high value”, “repeat customer”)
* Export contacts to CSV
* Role-based login (admin vs user)

GOAL:
This is not just a form viewer. It must function as a real CRM that helps businesses track leads, follow up, and increase conversions.

Keep the system simple, fast, and scalable. Prioritize functionality over visual complexity.
